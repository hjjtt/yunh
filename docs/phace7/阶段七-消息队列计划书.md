## 📚 核心概念

### 1. 为什么需要消息队列？

#### 场景：用户选课成功后的后续操作

```
┌─────────────────────────────────────────────────────────────┐
│                    同步调用方案（有问题）                    │
│                                                              │
│  用户选课 → [User 服务] ──→ 发送短信通知                     │
│              │            ──→ 发送邮件通知                   │
│              │            ──→ 记录操作日志                   │
│              │            ──→ 推送学习提醒                   │
│              │            ──→ 更新推荐系统                   │
│              │                                              │
│  ❌ 问题：                                            │
│  1. 响应时间长：所有操作完成才能返回                        │
│  2. 耦合严重：User 服务依赖太多其他服务                     │
│  3. 可靠性差：某个服务失败影响主流程                        │
│  4. 难以扩展：新增通知类型需要改 User 服务代码               │
└─────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────┐
│                    消息队列方案（推荐）                      │
│                                                              │
│  用户选课 → [User 服务] ──→ 发送消息到 MQ                    │
│              │                   │                          │
│              │                   ▼                          │
│              │            ┌─────────────┐                   │
│              │            │   RocketMQ  │                   │
│              │            └─────────────┘                   │
│              │                   │                          │
│              │         ┌─────────┼─────────┐                │
│              │         │         │         │                │
│              ▼         ▼         ▼         ▼                │
│         [短信服务] [邮件服务] [日志服务] [推荐服务]          │
│                                                              │
│  ✅ 优点：                                                   │
│  1. 响应快：发消息立即返回                                   │
│  2. 低耦合：User 服务不知道谁消费消息                        │
│  3. 高可靠：消费失败可重试                                   │
│  4. 易扩展：新增消费者不需要改 User 服务代码                  │
└─────────────────────────────────────────────────────────────┘
```

### 2. MQ 三大核心作用

| 作用 | 说明 | 场景 |
|------|------|------|
| **解耦** | 生产者和消费者互不依赖 | 选课成功后通知多个服务 |
| **异步** | 不等待耗时操作完成 | 发送邮件、短信、报表 |
| **削峰** | 缓冲突发流量 | 秒杀、抢购、热点事件 |

---

### 3. 主流 MQ 对比

| 特性 | RocketMQ | Kafka | RabbitMQ |
|------|----------|-------|----------|
| **开发语言** | Java | Scala | Erlang |
| **单机吞吐** | 10 万 + | 100 万 + | 万级 |
| **延迟** | 毫秒级 | 毫秒级 | 微秒级 |
| **可靠性** | 高（支持事务消息） | 中 | 高 |
| **消息堆积** | 支持（百万级） | 支持 | 不支持 |
| **适用场景** | 电商、金融、订单 | 日志收集、大数据 | 中小规模、低延迟 |

**本项目选择：RocketMQ**
- ✅ 阿里巴巴出品，中文文档完善
- ✅ 支持事务消息（分布式事务最终一致性）
- ✅ 消息堆积能力强
- ✅ 支持延迟消息、顺序消息

### 4. RocketMQ 架构

```
┌─────────────────────────────────────────────────────────────┐
│                      RocketMQ 架构                           │
│                                                              │
│  ┌─────────────┐         ┌─────────────┐                    │
│  │  Producer   │         │   Consumer  │                    │
│  │  (生产者)   │         │   (消费者)   │                    │
│  └──────┬──────┘         └──────┬──────┘                    │
│         │                       │                            │
│         │  发送消息              │  消费消息                  │
│         │                       │                            │
│         ▼                       ▼                            │
│  ┌─────────────────────────────────────────┐                 │
│  │              NameServer                  │                 │
│  │          (注册中心，无状态)               │                 │
│  └─────────────────────────────────────────┘                 │
│         │                       │                            │
│         │  路由信息              │  路由信息                  │
│         │                       │                            │
│         ▼                       ▼                            │
│  ┌─────────────────────────────────────────┐                 │
│  │                 Broker                   │                 │
│  │          (消息存储，核心组件)             │                 │
│  │  ┌───────────┐  ┌───────────┐           │                 │
│  │  │  Queue 1  │  │  Queue 2  │  ...      │                 │
│  │  └───────────┘  └───────────┘           │                 │
│  └─────────────────────────────────────────┘                 │
└─────────────────────────────────────────────────────────────┘

核心概念：
- Topic：消息主题（如：course_order_topic）
- Tag：消息标签（如：order_create、order_pay）
- Queue：消息队列（Topic 的分区，支持并发）
- Group：消费者组（同一组消费者负载均衡）
- Message：消息实体（包含 Body、Key、Tag 等）
```

---

### 5. 消息发送方式

| 方式 | 说明 | 场景 | 代码示例 |
|------|------|------|---------|
| **同步发送** | 等待 Broker 确认 | 重要消息（订单、支付） | `send(msg)` |
| **异步发送** | 回调接收结果 | 耗时敏感场景 | `send(msg, callback)` |
| **单向发送** | 不等待确认 | 日志、监控 | `sendOneway(msg)` |

---

### 6. 消息消费模式

| 模式 | 说明 | 场景 |
|------|------|------|
| **集群消费** | 消息只被组内一个消费者消费 | 大多数场景（默认） |
| **广播消费** | 消息被组内所有消费者消费 | 配置更新、缓存刷新 |

#### 7.4.3 实现消息发送服务

```java
// yunh-service-user/src/main/java/.../service/MqMessageService.java
package com.yunh.user.service;

import com.yunh.basic.constant.MqConstant;
import com.yunh.pojo.dto.CourseOrderMessage;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.apache.rocketmq.spring.core.RocketMQTemplate;
import org.springframework.messaging.Message;
import org.springframework.messaging.support.MessageBuilder;
import org.springframework.stereotype.Service;

/**
 * MQ 消息发送服务
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class MqMessageService {
    
    private final RocketMQTemplate rocketMQTemplate;
    
    /**
     * 同步发送消息（重要消息，需要确认）
     * 
     * @param topic Topic
     * @param tag Tag
     * @param message 消息体
     * @return 是否发送成功
     */
    public boolean sendSync(String topic, String tag, CourseOrderMessage message) {
        try {
            String destination = topic + ":" + tag;
            Message<CourseOrderMessage> msg = MessageBuilder
                .withPayload(message)
                .build();
            
            var sendResult = rocketMQTemplate.syncSend(destination, msg);
            
            log.info("同步发送消息成功：topic={}, tag={}, messageId={}, orderId={}",
                topic, tag, sendResult.getMsgId(), message.getOrderId());
            
            return sendResult.getSendStatus().equals(org.apache.rocketmq.client.producer.SendStatus.SEND_OK);
            
        } catch (Exception e) {
            log.error("同步发送消息失败：topic={}, tag={}, orderId={}",
                topic, tag, message.getOrderId(), e);
            return false;
        }
    }
    
    /**
     * 异步发送消息（耗时敏感场景）
     */
    public void sendAsync(String topic, String tag, CourseOrderMessage message) {
        String destination = topic + ":" + tag;
        Message<CourseOrderMessage> msg = MessageBuilder
            .withPayload(message)
            .build();
        
        rocketMQTemplate.asyncSend(destination, msg, new org.apache.rocketmq.client.producer.SendCallback() {
            @Override
            public void onSuccess(org.apache.rocketmq.client.producer.SendResult sendResult) {
                log.info("异步发送消息成功：topic={}, tag={}, messageId={}",
                    topic, tag, sendResult.getMsgId());
            }
            
            @Override
            public void onException(Throwable e) {
                log.error("异步发送消息失败：topic={}, tag={}, orderId={}",
                    topic, tag, message.getOrderId(), e);
                // TODO: 记录失败消息，后续补偿
            }
        });
    }
    
    /**
     * 单向发送消息（日志等不重要消息）
     */
    public void sendOneway(String topic, String tag, CourseOrderMessage message) {
        String destination = topic + ":" + tag;
        Message<CourseOrderMessage> msg = MessageBuilder
            .withPayload(message)
            .build();
        
        rocketMQTemplate.sendOneWay(destination, msg);
        log.debug("单向发送消息：topic={}, tag={}, orderId={}", topic, tag, message.getOrderId());
    }
    
    /**
     * 发送选课成功消息
     */
    public void sendCourseOrderSuccess(CourseOrderMessage message) {
        sendSync(MqConstant.TOPIC_COURSE_ORDER, MqConstant.TAG_ORDER_CREATE, message);
    }
    
    /**
     * 发送延迟消息（如：30 分钟后检查订单支付状态）
     * 
     * @param delayLevel 延迟级别（1s 5s 10s 30s 1m 2m 3m 4m 5m 6m 7m 8m 9m 10m 20m 30m 1h 2h）
     *                   对应 1-18
     */
    public void sendDelayMessage(String topic, String tag, CourseOrderMessage message, int delayLevel) {
        String destination = topic + ":" + tag;
        Message<CourseOrderMessage> msg = MessageBuilder
            .withPayload(message)
            .build();
        
        rocketMQTemplate.syncSend(destination, msg, 3000, delayLevel);
        log.info("发送延迟消息：topic={}, tag={}, delayLevel={}, orderId={}",
            topic, tag, delayLevel, message.getOrderId());
    }
}
```

// yunh-service-user/src/main/java/.../service/impl/UserCourseServiceImpl.java
package com.yunh.user.service.impl;

import com.yunh.api.client.CourseClient;
import com.yunh.pojo.dto.CourseOrderMessage;
import com.yunh.pojo.dto.SelectCourseDTO;
import com.yunh.pojo.entity.UserCourse;
import com.yunh.user.mapper.UserCourseMapper;
import com.yunh.user.service.MqMessageService;
import io.seata.spring.annotation.GlobalTransactional;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Slf4j
@Service
@RequiredArgsConstructor
public class UserCourseServiceImpl implements UserCourseService {
    
    private final UserCourseMapper userCourseMapper;
    private final CourseClient courseClient;
    private final MqMessageService mqMessageService;
    
    @Override
    @GlobalTransactional(name = "yunh-select-course", rollbackFor = Exception.class)
    @Transactional(rollbackFor = Exception.class)
    public void selectCourse(SelectCourseDTO dto) {
        Long userId = dto.getUserId();
        Long courseId = dto.getCourseId();
        
        log.info("开始选课事务：userId={}, courseId={}", userId, courseId);
        
        // 1. 添加选课记录
        UserCourse userCourse = new UserCourse();
        userCourse.setUserId(userId);
        userCourse.setCourseId(courseId);
        userCourse.setStatus(1); // 已选课
        userCourseMapper.insert(userCourse);
        
        // 2. 扣减课程库存
        courseClient.decreaseStock(courseId, 1);
        
        // 3. 构建消息体
        CourseOrderMessage message = new CourseOrderMessage();
        message.setOrderId(userCourse.getId());
        message.setUserId(userId);
        message.setCourseId(courseId);
        message.setCourseName("Spring Cloud 微服务实战"); // TODO: 从数据库获取
        message.setAmount(java.math.BigDecimal.valueOf(199.00));
        message.setStatus(1);
        message.setCreateTime(new java.util.Date());
        
        // 4. 发送选课成功消息
        mqMessageService.sendCourseOrderSuccess(message);
        
        log.info("选课事务完成：userId={}, courseId={}", userId, courseId);
    }
}

// yunh-service-user/src/main/java/.../mq/consumer/SmsNotificationConsumer.java
package com.yunh.user.mq.consumer;

import com.yunh.basic.constant.MqConstant;
import com.yunh.pojo.dto.CourseOrderMessage;
import lombok.extern.slf4j.Slf4j;
import org.apache.rocketmq.spring.annotation.RocketMQMessageListener;
import org.apache.rocketmq.spring.core.RocketMQListener;
import org.springframework.stereotype.Service;

/**
 * 短信通知消费者
 */
@Slf4j
@Service
@RocketMQMessageListener(

#### 7.6.1 事务消息监听器

```java
// yunh-service-user/src/main/java/.../mq/transaction/CourseOrderTransactionListener.java
package com.yunh.user.mq.transaction;

import com.yunh.pojo.dto.CourseOrderMessage;
import com.yunh.user.service.UserCourseService;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.apache.rocketmq.spring.annotation.RocketMQTransactionListener;
import org.apache.rocketmq.spring.core.RocketMQLocalTransactionListener;
import org.apache.rocketmq.spring.core.RocketMQLocalTransactionState;
import org.springframework.messaging.Message;
import org.springframework.stereotype.Component;

import java.util.concurrent.ConcurrentHashMap;

/**
 * 选课订单事务消息监听器
 * 
 * 事务消息流程：
 * 1. 发送半消息（Half Message）到 Broker
 * 2. 执行本地事务
 * 3. 根据本地事务结果提交/回滚消息
 * 4. 如果步骤 3 失败，Broker 会主动回查事务状态
 */
@Slf4j
@Component
@RequiredArgsConstructor
@RocketMQTransactionListener(txProducerGroup = "yunh-transaction-producer-group")
public class CourseOrderTransactionListener implements RocketMQLocalTransactionListener {
    
    /**
     * 本地事务执行表（实际项目中应该用数据库）
     * key: 事务 ID, value: 事务状态
     */
    private final ConcurrentHashMap<String, Integer> localTransactionTable = new ConcurrentHashMap<>();
    
    private final UserCourseService userCourseService;
    
    /**
     * 执行本地事务
     * 
     * @param message 消息
     * @param extension 扩展信息
     * @return 事务状态（COMMIT/ROLLBACK/UNKNOWN）
     */
    @Override
    public RocketMQLocalTransactionState executeLocalTransaction(Message<?> message, Object extension) {
        try {
            CourseOrderMessage orderMessage = (CourseOrderMessage) 
                ((org.springframework.messaging.support.GenericMessage<?>) message).getPayload();
            
            log.info("开始执行本地事务：orderId={}", orderMessage.getOrderId());
            
            // TODO: 执行本地事务（如创建订单）
            // userCourseService.createOrder(orderMessage);
            
            // 记录事务状态
            String transactionId = orderMessage.getOrderId().toString();
            localTransactionTable.put(transactionId, 1); // 1 = 成功
            
            log.info("本地事务执行成功：orderId={}", orderMessage.getOrderId());
            
            return RocketMQLocalTransactionState.COMMIT;
            
        } catch (Exception e) {
            log.error("本地事务执行失败", e);
            return RocketMQLocalTransactionState.ROLLBACK;
        }
    }
    
    /**
     * 事务状态回查
     * 
     * 当 Broker 未收到提交/回滚确认时，会主动回查事务状态
     * 
     * @param message 消息
     * @return 事务状态
     */
    @Override
    public RocketMQLocalTransactionState checkLocalTransaction(Message<?> message) {
        try {
            CourseOrderMessage orderMessage = (CourseOrderMessage) 
                ((org.springframework.messaging.support.GenericMessage<?>) message).getPayload();
            
            String transactionId = orderMessage.getOrderId().toString();
            Integer status = localTransactionTable.get(transactionId);
            
            log.info("事务状态回查：orderId={}, status={}", orderMessage.getOrderId(), status);
            
            if (status == null) {
                return RocketMQLocalTransactionState.ROLLBACK;
            } else if (status == 1) {
                return RocketMQLocalTransactionState.COMMIT;
            } else {
                return RocketMQLocalTransactionState.ROLLBACK;
            }
            
        } catch (Exception e) {
            log.error("事务状态回查失败", e);
            return RocketMQLocalTransactionState.ROLLBACK;
        }
    }
}
```

package com.yunh.user.service;

import com.yunh.basic.constant.MqConstant;
import com.yunh.pojo.dto.CourseOrderMessage;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.apache.rocketmq.spring.core.RocketMQTemplate;
import org.springframework.messaging.Message;
import org.springframework.messaging.support.MessageBuilder;
import org.springframework.stereotype.Service;

@Slf4j
@Service
@RequiredArgsConstructor
public class MqMessageService {
    
    private final RocketMQTemplate rocketMQTemplate;
    
    /**
     * 发送事务消息
     * 
     * @param topic Topic
     * @param tag Tag
     * @param message 消息体
     * @param transactionId 事务 ID
     */
    public void sendTransactionMessage(String topic, String tag, 
                                       CourseOrderMessage message, String transactionId) {
        String destination = topic + ":" + tag;
        
        Message<CourseOrderMessage> msg = MessageBuilder
            .withPayload(message)
            .setHeader(org.apache.rocketmq.common.message.MessageConst.PROPERTY_UNIQ_CLIENT_MESSAGE_ID_KEYIDX, transactionId)
            .build();
        
        // 发送事务消息
        rocketMQTemplate.sendMessageInTransaction(destination, msg, null);
        
        log.info("事务消息已发送：topic={}, tag={}, transactionId={}, orderId={}",
            topic, tag, transactionId, message.getOrderId());
    }
    
    /**
     * 发送选课订单事务消息
     */
    public void sendCourseOrderTransaction(CourseOrderMessage message) {
        String transactionId = "order_" + message.getOrderId() + "_" + System.currentTimeMillis();
        sendTransactionMessage(MqConstant.TOPIC_COURSE_ORDER, MqConstant.TAG_ORDER_CREATE, message, transactionId);
    }
}

#### 7.7.1 消息不丢失方案

| 环节 | 措施 | 配置 |
|------|------|------|
| **生产者** | 同步发送 + 重试 | `retry-times-when-send-failed: 2` |
| **Broker** | 同步刷盘 + 同步复制 | `flushDiskType=SYNC_FLUSH` |
| **消费者** | 消费成功再确认 | 默认自动 ACK，失败抛异常 |

## ⚠️ 注意事项

### 1. 版本兼容性

| 组件 | 版本 |
|------|------|
| RocketMQ | 4.9.4 |
| rocketmq-spring-boot-starter | 2.2.3 |

### 2. 消息丢失风险

| 场景 | 解决方案 |
|------|---------|
| 生产者发送失败 | 同步发送 + 重试 |
| Broker 宕机 | 同步刷盘 + 主从复制 |
| 消费者消费失败 | 失败重试 + 死信队列 |

### 3. 消息重复消费

**原因：** 网络抖动、消费者宕机、ACK 丢失

**解决方案：**
- 业务层实现幂等性
- 使用唯一键（订单 ID）去重
- Redis 记录已消费消息

### 4. 消息堆积

**监控指标：** 消费者堆积量 > 10000 需要告警

**解决方案：**
- 增加消费者实例
- 优化消费逻辑
- 扩容 Broker

### 5. 顺序消息

**场景：** 订单状态变更（创建→支付→发货）

**解决方案：**
```java
// 发送顺序消息
rocketMQTemplate.syncSendOrderly(destination, msg, hashKey);
```
