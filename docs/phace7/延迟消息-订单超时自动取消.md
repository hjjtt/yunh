# 延迟消息 —— 订单超时自动取消功能记录

> 项目：云课堂（YunH）
> 技术栈：Spring Boot 2.2.5 + Spring Cloud Hoxton.SR3 + Spring Cloud Alibaba 2.2.1 + RocketMQ 4.9.4 + Seata 1.4.2

---

## 一、功能概述

用户选课下单后，如果在指定时间内未完成支付，系统通过 **RocketMQ 延迟消息** 自动取消订单并恢复课程库存。

### 核心思路

```
用户选课下单 ──→ 发送延迟消息（30分钟后投递）──→ RocketMQ 暂存消息
                                                        │
                                                   30分钟后投递
                                                        │
                                                        ▼
                                               消费者收到消息
                                                   │
                                          ┌────────┴────────┐
                                          ▼                 ▼
                                     status=0            status=1
                                     (未支付)            (已支付)
                                          │                 │
                                     自动取消+恢复库存    忽略，不处理
```

### 订单状态流转

| 状态值 | 含义 | 说明 |
|--------|------|------|
| 0 | 待支付 | 用户选课后的初始状态 |
| 1 | 已支付 | 用户在超时前完成支付 |
| 2 | 已取消 | 超时未支付，系统自动取消 |

---

## 二、RocketMQ 延迟级别对照

RocketMQ 开源版支持 18 个固定延迟级别，不支持任意时间延迟：

| 级别 | 延迟时间 | 级别 | 延迟时间 |
|------|----------|------|----------|
| 1 | 1 秒 | 10 | 6 分钟 |
| 2 | 5 秒 | 11 | 7 分钟 |
| **3** | **10 秒** | 12 | 8 分钟 |
| 4 | 30 秒 | 13 | 9 分钟 |
| 5 | 1 分钟 | 14 | 10 分钟 |
| 6 | 2 分钟 | 15 | 20 分钟 |
| 7 | 3 分钟 | **16** | **30 分钟** |
| 8 | 4 分钟 | 17 | 1 小时 |
| 9 | 5 分钟 | 18 | 2 小时 |

- **生产环境**：使用级别 16（30 分钟）
- **开发测试**：使用级别 3（10 秒），便于快速验证

### 4.3 延迟消息发送 —— MqMessageService.sendDelayMessage()

```java
public void sendDelayMessage(String topic, String tag, CourseOrderMessage message, int delayLevel) {
    String destination = topic + ":" + tag;
    Message<CourseOrderMessage> msg = MessageBuilder
            .withPayload(message)
            .build();

    // 核心API：syncSend(destination, msg, timeout, delayLevel)
    rocketMQTemplate.syncSend(destination, msg, 3000, delayLevel);
}
```

**关键参数说明**：
- `destination`：格式为 `topic:tag`，例如 `course_order_topic:order_cancel`
- `timeout`：发送超时时间，默认 3000ms
- `delayLevel`：延迟级别，对应 RocketMQ 的 18 个固定等级

### 4.4 选课下单（发送延迟消息）—— UserCourseService.selectCourse()

```java
@GlobalTransactional(name = "yunh-select-course", rollbackFor = Exception.class)
@Transactional(rollbackFor = Exception.class)
public Result<Void> selectCourse(SelectCourseDTO dto) {
    Long userId = dto.getUserId();
    Long courseId = dto.getCourseId();

    // 步骤 0：检查是否已选过该课程（避免唯一索引冲突）
    Integer activeCount = userCourseMapper.selectCount(
            new QueryWrapper<UserCourse>()
                    .eq("user_id", userId)
                    .eq("course_id", courseId)
                    .ne("status", 2));
    if (activeCount > 0) {
        return Result.error("已选过该课程，请勿重复选课");
    }

    // 步骤 0.1：清理已取消的旧记录，允许重新选课
    userCourseMapper.delete(
            new QueryWrapper<UserCourse>()
                    .eq("user_id", userId)
                    .eq("course_id", courseId)
                    .eq("status", 2));

    // 步骤 1：添加选课记录（本地事务，status=0 待支付）
    UserCourse userCourse = new UserCourse();
    userCourse.setUserId(userId);
    userCourse.setCourseId(courseId);
    userCourse.setStatus(0);
    userCourse.setPayType(dto.getPayType());
    userCourseMapper.insert(userCourse);

    // 步骤 2：扣减课程库存（远程调用 Course 服务，失败则全局回滚）
    courseFeignClient.decreaseStock(courseId, 1);

    // 步骤 3：发送选课成功通知消息
    CourseOrderMessage message = new CourseOrderMessage();
    message.setOrderId(userCourse.getId());
    message.setUserId(userId);
    message.setCourseId(courseId);
    // ... 设置其他字段
    mqMessageService.sendCourseOrderSuccess(message);

    // 步骤 4：发送延迟消息（30分钟后检查订单状态）
    mqMessageService.sendDelayMessage(
            MqConstant.TOPIC_COURSE_ORDER,
            MqConstant.TAG_ORDER_CANCEL,
            message,
            MqConstant.DELAY_LEVEL_ORDER_TIMEOUT);

    return Result.success();
}
```

**事务保障**：
- `@GlobalTransactional`（Seata）：保证步骤 1 和步骤 2 的分布式事务一致性
- 步骤 2 失败 → 步骤 1 自动回滚
- 步骤 3、4 是 MQ 消息，在事务提交后异步处理

### 4.5 延迟消息消费者 —— OrderTimeoutConsumer.java

```java
@Slf4j
@Service
@RequiredArgsConstructor
@RocketMQMessageListener(
        topic = MqConstant.TOPIC_COURSE_ORDER,
        selectorExpression = MqConstant.TAG_ORDER_CANCEL,
        consumerGroup = MqConstant.GROUP_ORDER_TIMEOUT_CONSUMER
)
public class OrderTimeoutConsumer implements RocketMQListener<CourseOrderMessage> {

    private final UserCourseService userCourseService;

    @Override
    public void onMessage(CourseOrderMessage message) {
        log.info("收到订单超时检查消息：orderId={}", message.getOrderId());

        try {
            boolean cancelled = userCourseService.cancelTimeoutOrder(message.getOrderId());

            if (cancelled) {
                log.info("订单超时未支付，已自动取消：orderId={}", message.getOrderId());
            } else {
                log.info("订单已支付或已取消，无需处理：orderId={}", message.getOrderId());
            }
        } catch (Exception e) {
            log.error("订单超时取消处理失败：orderId={}", message.getOrderId(), e);
            throw new RuntimeException("订单超时取消失败", e);
        }
    }
}
```

**注解说明**：
- `@RocketMQMessageListener`：声明式消费者，Spring Boot 自动注册
- `topic`：监听的 Topic
- `selectorExpression`：Tag 过滤表达式，只消费 `order_cancel` 标签的消息
- `consumerGroup`：消费者组名，同组内的消费者负载均衡消费

### 4.6 超时取消逻辑 —— UserCourseService.cancelTimeoutOrder()

```java
@GlobalTransactional(name = "yunh-cancel-timeout-order", rollbackFor = Exception.class)
@Transactional(rollbackFor = Exception.class)
public boolean cancelTimeoutOrder(Long orderId) {
    UserCourse userCourse = userCourseMapper.selectById(orderId);
    if (userCourse == null) {
        return false;
    }

    // 已支付或已取消，不处理
    if (userCourse.getStatus() != 0) {
        return false;
    }

    // 取消订单：更新状态为 2（已取消）
    UserCourse update = new UserCourse();
    update.setId(orderId);
    update.setStatus(2);
    update.setUpdateTime(LocalDateTime.now());
    userCourseMapper.updateById(update);

    // 恢复课程库存（远程调用 Course 服务）
    Result<Void> stockResult = courseFeignClient.increaseStock(userCourse.getCourseId(), 1);
    if (stockResult == null || !stockResult.isSuccess()) {
        throw new RuntimeException("恢复课程库存失败：Course 服务返回错误");
    }

    return true;
}
```

**注意事项**：
- 必须检查 Feign 返回值的 `isSuccess()`，Feign 降级不会抛异常，而是返回 `Result.error()`
- 使用 `@GlobalTransactional` 保证取消订单 + 恢复库存的分布式事务一致性

## 七、踩坑记录与解决方案

### 7.1 DuplicateKeyException：唯一索引冲突

**现象**：
```
DuplicateKeyException: Duplicate entry '1-1' for key 't_user_course.uk_user_course'
```

**原因**：
- 表 `t_user_course` 有唯一索引 `uk_user_course(user_id, course_id)`
- 订单取消后（status=2），记录仍占用唯一索引
- 用户再次选课，INSERT 时触发唯一索引冲突

**解决方案**：
1. 选课前先检查是否已有活跃记录（status != 2）
2. 清理已取消的旧记录（DELETE status=2），允许重新选课

```java
// 检查重复选课
Integer activeCount = userCourseMapper.selectCount(
        new QueryWrapper<UserCourse>()
                .eq("user_id", userId)
                .eq("course_id", courseId)
                .ne("status", 2));
if (activeCount > 0) {
    return Result.error("已选过该课程，请勿重复选课");
}

// 清理已取消的旧记录
userCourseMapper.delete(
        new QueryWrapper<UserCourse>()
                .eq("user_id", userId)
                .eq("course_id", courseId)
                .eq("status", 2));
```

### 7.2 Feign 降级返回值未检查

**现象**：
- Course 服务未启动 → Feign 降级触发 → 返回 `Result.error()`
- 但 `cancelTimeoutOrder()` 没有检查返回值，误打印 "课程库存已恢复"
- 实际库存并未恢复

**解决方案**：必须检查 `Result.isSuccess()`

```java
Result<Void> stockResult = courseFeignClient.increaseStock(courseId, 1);
if (stockResult == null || !stockResult.isSuccess()) {
    throw new RuntimeException("恢复课程库存失败");
}
```

### 7.3 selectCount 返回值类型

**现象**：编译错误 `Integer cannot convert to Long`

**原因**：MyBatis-Plus 的 `selectCount()` 返回 `Integer`，不是 `Long`

**解决**：使用 `Integer` 接收返回值

```java
// ❌ 错误
Long existCount = userCourseMapper.selectCount(...);

// ✅ 正确
Integer existCount = userCourseMapper.selectCount(...);
```

### 7.4 PowerShell 发送 JSON 请求

**现象**：`curl.exe` 在 PowerShell 中转义 `\"` 失败，JSON 解析报错

**解决方案**：使用 PowerShell 原生 `Invoke-RestMethod`

```powershell
# ❌ curl.exe 转义问题
curl.exe -X POST ... -d '{\"userId\":1}'

# ✅ 使用 Invoke-RestMethod
$body = '{"userId":1,"courseId":1,"payType":1}'
Invoke-RestMethod -Uri "http://localhost:9000/api/yunh-service-user/user/course/select" `
    -Method Post -ContentType "application/json" -Body $body
```
