## ⚠️ 注意事项

### 1. 开发顺序建议

```
第一阶段（基础）：
Auth → User → Course → Gateway

第二阶段（核心）：
Order → Pay → Video

第三阶段（增强）：
Interaction → Search → Statistics

第四阶段（部署）：
Docker → K8s → CI/CD → 监控
```

### 2. 数据库设计原则

- 每个服务独立数据库
- 禁止跨库 JOIN
- 通过 Feign/MQ 进行数据交互
- 冗余必要字段减少调用

### 3. 接口设计规范

- RESTful 风格
- 统一响应格式
- 统一错误码
- 完善的 API 文档

### 4. 安全注意事项

- 密码加密存储（BCrypt）
- Token 有效期控制
- 敏感操作二次验证
- SQL 注入防护
- XSS 攻击防护
