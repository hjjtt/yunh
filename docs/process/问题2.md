## 一、代理 1：前端设计美观度 — 评分 7.0/10

### 核心问题：学员端与教师端风格分裂

| 维度 | 教师端 | 学员端 |
|------|--------|--------|
| CSS 变量使用 | 完美 | 大量硬编码 |
| 页面均分 | 9/10 | 5-6/10 |
| 空状态设计 | 统一圆形图标 | 各自实现，4 种不同样式 |

### 各页面评分

| 页面 | 分数 | 主要问题 |
|------|------|----------|
| pagesTeacher/* (4 页) | 9/10 | 标杆页面，风格统一 |
| packageTeacher/course-edit | 9/10 | 表单设计优秀 |
| comment-create / question-create | 8/10 | 表单规范，变量使用好 |
| comment-detail / question-detail | 8/10 | 采纳徽章/删除按钮有非系统色 |
| course/list | 7/10 | 基本遵循，少量硬编码 |
| search/index | 6/10 | 英雄区好，内容区硬编码 |
| home/index | 6/10 | metric-card 大圆角 + 橙色渐变脱离风格 |
| my-notes / comments / questions | 6/10 | 硬编码 #0f172a / #94a3b8 |
| note-detail | 5/10 | 删除按钮用 Tailwind 色系 |
| auth/login | 5/10 | 完全脱离设计系统 |
| profile/index | 5/10 | 大圆角、无装订边、引入新色系 |
| player | 4/10 | 设计最简陋，无卡片包装 |

### Top 5 设计改进建议

1. **全局颜色变量替换** — `#0f172a` → `var(--ink-dark)`，`#94a3b8` → `var(--ink-light)` 等，涉及 70+ 处
2. **重构 login 页** — 背景改纸张色，圆角统一 4rpx，添加复古元素
3. **统一圆角系统** — 定义 `--radius-sm` / `--radius-lg`，消除 8rpx / 12rpx / 28rpx 等不一致
4. **提升 player 页** — 信息区用 `.card` 包装，添加面包屑、空状态图标
5. **统一空状态组件** — 定义全局 `.empty-state` 系列类，替换各页面 4 种不同实现

## 二、代理 2：前后端 API 一致性

### 匹配概况

- 前端 **39** 个 API 调用，**37** 个与后端完全匹配
- 后端 **56** 个接口，**24** 个前端未调用（多为管理端，合理）
- Gateway **14** 条路由全部正确

### 问题清单

| 级别 | 问题 | 影响 |
|------|------|------|
| 严重 | 缺少 `services/search.js`，搜索功能未对接 | 搜索页无法工作 |
| 严重 | 缺少购物车 Cart 前端调用 | 4 个后端接口闲置 |
| 中等 | `teacher.js` 和 `video.js` 重复定义 `getChapters` | 维护隐患 |
| 中等 | `decreaseStock` 接口暴露给前端 | 安全风险，应仅内部调用 |
| 中等 | 金额通过 Query String 传递 | 精度丢失风险 |
| 中等 | `payCallback` 硬编码模拟支付号 | 测试代码混入生产 |
| 轻微 | `refreshAccessToken` 不在 services 层 | 架构不规范 |
