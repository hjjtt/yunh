# 服务启动配置修改操作总结

本文档记录本次解决服务启动问题的**具体操作步骤和修改内容**，可直接复现。

---

## 📋 修改前问题总览
1. 服务启动时Nacos 401认证失败，无法拉取配置
2. 启动日志存在多个不影响功能但不美观的警告
3. 配置文件中存在错误的Nacos地址和禁用配置

---

## 🔧 具体修改操作（按文件分类）

### 一、修改 `yunh-service-user/src/main/resources/bootstrap.yml`
#### 修改前（错误配置）
```yaml
spring:
  application:
    name: yunh-service-user
  profiles:
    active: dev  # 激活环境：dev/test/prod
  cloud:
    nacos:
      discovery:
        enabled: false                  # 禁用服务注册
        server-addr: 10.150.9.193:8080  # 错误地址
        namespace: public
      config:
        enabled: false                  # 禁用配置中心
        server-addr: 10.150.9.193:8080  # 错误地址
        file-extension: yml
        group: DEFAULT_GROUP
```

#### 修改后（正确配置）
```yaml
spring:
  application:
    name: yunh-service-user
#  profiles:
#    active: dev # 注释dev环境配置，避免查找不存在的dev后缀配置
  cloud:
    nacos:
      discovery:
        enabled: true                  # 启用服务注册
        server-addr: 127.0.0.1:8848    # 修正为本地Nacos正确地址
        namespace: public
        username: your_nacos_username                # 添加Nacos账号
        password: your_password_here               # 添加Nacos密码
      config:
        enabled: true                  # 启用配置中心
        server-addr: 127.0.0.1:8848    # 修正为本地Nacos正确地址
        file-extension: yml
        group: DEFAULT_GROUP
        username: your_nacos_username                # 添加配置中心账号
        password: your_password_here               # 添加配置中心密码
```

---

### 二、修改 `yunh-service-course/src/main/resources/bootstrap.yml`
#### 修改内容与上述user服务完全一致：
1. 注释dev环境配置
2. 启用Nacos服务注册和配置中心
3. 修正Nacos地址为`127.0.0.1:8848`
4. 添加Nacos账号密码配置

---

### 三、修改 `yunh-service-user/src/main/resources/application.yml`
#### 修改内容：
1. **在logging.level下添加一行配置**，屏蔽Actuator端点警告：
   ```yaml
   org.springframework.boot.actuate.endpoint: ERROR
   ```
2. **暂时注释MyBatis Plus配置**（后续开发Mapper时可取消注释）：
   ```yaml
   # mybatis-plus:
   #   mapper-locations: classpath*:/mapper/**/*.xml
   #   type-aliases-package: com.yunh.user.pojo
   #   configuration:
   #     map-underscore-to-camel-case: true
   #     log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
   ```

---

### 四、修改 `yunh-service-course/src/main/resources/application.yml`
#### 修改内容与上述user服务完全一致：
1. 添加Actuator日志级别配置屏蔽警告
2. 暂时注释MyBatis Plus配置

---

## ✅ 修改后效果验证
1. **Nacos认证问题解决**：服务可以正常连接Nacos，成功拉取配置，不再报401错误
2. **所有警告消除**：启动日志干净，无任何警告信息
3. **服务注册成功**：启动后自动注册到Nacos服务列表，可在Nacos控制台查看
4. **服务正常运行**：接口可以正常调用，功能不受影响

---

## 📝 后续恢复操作
当需要开发数据库功能时，只需要取消application.yml中MyBatis Plus配置的注释，创建对应的Mapper接口即可正常使用。
