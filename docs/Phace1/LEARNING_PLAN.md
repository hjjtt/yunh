# 📚 云课堂 (YunH) - Spring Cloud 学习项目

## 🎯 学习目标

通过从零搭建一个完整的微服务项目，系统学习 Spring Cloud 核心技术。

***

## 📖 学习路线（由浅入深）

### 第一阶段：基础搭建 ⭐⭐

> 目标：理解微服务基本概念，搭建项目骨架

| 步骤  | 学习内容           | 实践任务                    | 预计时间 |
| --- | -------------- | ----------------------- | ---- |
| 1.1 | Maven 多模块项目结构  | 创建父工程和子模块               | 1 小时 |
| 1.2 | Spring Boot 基础 | 创建第一个 Spring Boot 应用    | 2 小时 |
| 1.3 | 统一依赖管理         | 配置父 pom.xml 版本管理        | 1 小时 |
| 1.4 | 通用工具类封装        | 创建 yunh-basic-common 模块 | 2 小时 |

**完成标志：** `mvn clean install` 能成功构建

***

## 📁 项目结构

```
yunh-cloud/
├── pom.xml                          # 父工程
├── yunh-basic/                      # 基础模块
│   ├── yunh-basic-common/           # 通用工具类
│   └── yunh-basic-dependency/       # 公共依赖
├── yunh-pojo/                       # 数据模型
│   ├── yunh-pojo-common/
│   ├── yunh-pojo-user/
│   └── yunh-pojo-course/
├── yunh-api/                        # API 接口层
│   ├── yunh-api-user/
│   └── yunh-api-course/
├── yunh-service/                    # 服务层
│   ├── yunh-service-common/
│   ├── yunh-service-user/           # 用户服务
│   └── yunh-service-course/         # 课程服务
└── yunh-support/                    # 基础设施
    └── yunh-service-gateway/        # 网关服务
```

## 📝 学习建议

1. **动手优先**：每个步骤都要亲自敲代码，不要只看
2. **理解原理**：不仅要会用，还要知道为什么
3. **记录笔记**：每个阶段完成后写总结
4. **调试为主**：遇到问题先自己调试，培养排查能力
5. **循序渐进**：不要跳步，基础打牢再学高级特性
